Machine virtuelle

La machine virtuelle fonctionne via virtual box.

L'image contient une machine basé sur Ubuntu, ou sont installé différents logiciels neccessaires au cours :

Netbeans

Java 8

Maven

git

wildfly

PostgresSQL

etc...

Vous pouvez télécharger l'image ici  (4 Go).

Le login est : siad

Le password est : siad2015

Les accès sont uniformisés, c'est à dire que l'acces à postgreSQL répond aussi à cette identité. Vous accès root sur vos machines.

Données personnelle

La machine virtuelle est réinitialisé chaque nuit, donc les données que vous stockez dessus ne sont pas pérenne. Pensez à sauvegarder vos données sur votre espace personnelle.

Je vous encourage à sauvegarder vos code sources pour pouvoir les consulter ultérieurement.

Utiliser la machine virtuelle

La machine virtuelle vous est fourni avec un système de base de donnée PostgreSQL qui est lancé au démarrage.

Voici le étapes pour ouvrir un projet (prenons par exemple le TD 1 de JSF.

Selectionnez le projet lié au TD

Ouvrez le projet

La structure du projet est classique et repondant aux pattern Maven

...

Pour executer le'application nous utilisons le serveur wildfly

Faites un clic droit du le module ouvert et selectionner RUN ou DEBUG selon votre besoin.

Pour la première fois, NETBEANS vous demande de selectionner le serveur. Dans le drop menu, selectionner Wildfly Application Server

Au cas selectionnez l'item pour le rendre permanent (mais cela ne fonctionnera pas pour les VM executée sur les postes de la fac)
